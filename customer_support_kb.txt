Overview

If your account has been locked due to multiple failed login attempts, you will need to wait 15 minutes before attempting to sign in again or request a password reset as described above. Contacting Support
If you continue to experience issues accessing your account after completing the password reset, please contact the AcmeTech Customer Support team by submitting a ticket through the support portal at support.acmetechsolutions.com or by emailing support@acmetechsolutions.com. Title: API Rate Limits Guide

Article: Understanding AcmeTech API Rate Limits
Category: API and Integrations
Last Updated: March 2026

Overview
AcmeTech enforces API rate limits to ensure fair usage of the platform and to protect system stability for all customers. This article explains how rate limits work, how to check your current usage, how to handle rate limit errors, and how to request a rate limit increase.

Default Rate Limits
AcmeTech API rate limits are applied per customer organization. The default rate limits by plan tier are: Starter Plan: 500 requests per minute. Professional Plan: 1,000 requests per minute. Enterprise Plan: Customizable, with a default of 5,000 requests per minute configurable up to your contracted maximum. Rate limits apply to all API endpoints collectively. Specific high-throughput endpoints such as batch operations may have additional per-endpoint limits documented in the API reference.

How Rate Limiting Works
AcmeTech uses a sliding window rate limiting algorithm. Your request quota is replenished continuously rather than resetting at a fixed interval. This means that if you send 500 requests in the first 10 seconds of a minute, you are not allowed to send 500 more requests at the start of the next minute; instead, you regain capacity gradually as older requests move out of the sliding window.

Handling Rate Limit Errors
When your request rate exceeds the configured limit, the AcmeTech API returns an HTTP 429 Too Many Requests response. The response includes a Retry-After header indicating the number of seconds you should wait before retrying. Your integration should implement exponential backoff with jitter when encountering 429 responses. Do not immediately retry at full request volume after the Retry-After period; increase the delay progressively if rate limit errors persist. Monitoring your API usage against your rate limits proactively is the best way to avoid unexpected 429 errors in production.

Monitoring API Usage
Organization administrators can view real-time and historical API usage in the AcmeTech developer dashboard under Settings > API Usage. Usage data is updated every minute. Alerts can be configured to notify your team when API usage exceeds a specified percentage of your rate limit. Setting up usage alerts is strongly recommended for all integrations with high request volumes.

Requesting a Rate Limit Increase
Enterprise customers who require higher rate limits than their default allocation can request an increase through their dedicated account manager or by submitting a support ticket with the subject "Rate Limit Increase Request." Include your organization name, current limit, requested limit, and a brief description of your use case. Rate limit increases for Enterprise customers are typically approved within two business days.

Title: SSO Setup Guide

Article: Setting Up Single Sign-On for Your AcmeTech Organization
Category: Security and Integrations
Last Updated: February 2026

Overview
AcmeTech supports Single Sign-On (SSO) integration with corporate identity providers via SAML 2.0 and OpenID Connect (OIDC). Enabling SSO allows your users to sign in to AcmeTech using their existing corporate credentials, eliminating the need for separate AcmeTech passwords and enabling centralized access management. SSO setup is available on Professional and Enterprise plans.

Prerequisites
Before configuring SSO, ensure you have administrator access to both your corporate identity provider and your AcmeTech organization account. You will need your identity provider's metadata URL or XML file (for SAML) or your identity provider's discovery endpoint URL, client ID, and client secret (for OIDC). SSO configuration should be tested in a non-production environment before being activated for all users.

Configuring SAML 2.0 SSO
Navigate to Account Settings > Security > Single Sign-On in AcmeTech. Select SAML 2.0 as the SSO type. Download the AcmeTech Service Provider metadata from the configuration page and upload it to your identity provider to create the AcmeTech application. In AcmeTech, enter your identity provider's metadata URL or upload the metadata XML file. Configure the attribute mapping to specify which identity provider attributes correspond to AcmeTech's required user fields: email address, first name, and last name. Save the configuration and click "Test SSO Connection" to validate before enabling.

Configuring OIDC SSO
Navigate to Account Settings > Security > Single Sign-On and select OpenID Connect as the SSO type. Enter your identity provider's discovery endpoint URL, client ID, and client secret. Configure the required scopes: openid, email, and profile. Enter the redirect URI displayed on the AcmeTech configuration page into your identity provider's application settings. Save the configuration and test the SSO flow using the "Test SSO Connection" button.

Activating SSO for All Users
Once the SSO configuration is tested and confirmed working, activate it for your organization by toggling "Enforce SSO Login" in the SSO configuration settings. After SSO is enforced, users will be redirected to your identity provider's login page when they attempt to sign in to AcmeTech. Existing AcmeTech-native passwords will no longer be accepted. Ensure all users have active accounts in your identity provider before enforcing SSO to prevent lockouts.

Troubleshooting SSO Issues
Common SSO configuration errors include certificate mismatch between the identity provider and AcmeTech's expected signature validation settings, incorrect attribute mapping causing user provisioning failures, and redirect URI mismatches in OIDC configurations. If users are unable to complete the SSO login flow, check the error details displayed in the browser and compare them against the SSO troubleshooting guide in the AcmeTech developer documentation.

If you are using AcmeTech's native authentication, the invitation email will be sent from noreply@acmetechsolutions.com. The invitation expires after 72 hours; you can resend it from the Users section if needed.

Step 2: Assign Groups and Permissions
If your organization uses custom groups to manage access to specific projects, customers, or features, add the new team member to the appropriate groups before their first login. Navigate to Account Settings > Groups, select the relevant group, and add the new user. Group membership determines which data and features the user can access within their assigned role.

Step 3: Share Getting Started Resources
AcmeTech provides a Getting Started guide and video tutorial series accessible from the Help Center at help.acmetechsolutions.com. Share the following resources with new team members: the Getting Started Guide for their specific role (Member, Admin, or View Only), the AcmeTech Quick Start video (15 minutes), and the keyboard shortcuts reference card. The in-app onboarding checklist will appear automatically for new users on their first login and guides them through the platform's core features.

Step 4: Schedule a Walkthrough
For teams onboarding five or more new users at once, AcmeTech offers a complimentary onboarding webinar facilitated by a Customer Success team member. Professional and Enterprise plan customers can schedule an onboarding session by contacting their Customer Success Manager or emailing onboarding@acmetechsolutions.com. The session covers platform navigation, key workflows, and best practices relevant to your team's use case.

Step 5: Monitor Adoption
After onboarding, track new user adoption through the User Activity Report in Account Settings > Users > Reports. Users who have not completed the in-app onboarding checklist or who have not logged in within seven days of invitation can be identified in this report. Follow up with these users to address any barriers to adoption. AcmeTech's Customer Success team can provide additional enablement resources on request.

Title: Data Import Guide

Article: Importing Data into AcmeTech
Category: Data Management
Last Updated: April 2026

Overview
AcmeTech provides data import tools to help new customers migrate data from existing systems and to enable ongoing bulk data operations. This article describes the supported import types, file format requirements, and the step-by-step import process.

Supported Import Types
AcmeTech currently supports the following data imports. Customer Records: Import a list of customer organizations and contacts in CSV format. Bulk User Import: Add or update multiple AcmeTech users at once using a CSV file as described in the User Management guide. Historical Data Migration: Enterprise plan customers can work with the AcmeTech Professional Services team to migrate large volumes of historical data, including transactional records and attachments.

Preparing Your Import File
Download the import template for your data type from the Import Data section in Account Settings > Data Management > Import Data. The template includes all required and optional columns with descriptions and example values. Ensure your import file follows these requirements: UTF-8 encoding, comma-delimited CSV format, values containing commas enclosed in double quotes, and no extra rows or columns outside the template structure. Validate required fields are populated for every row. Common required fields include a unique identifier, name, and at minimum one contact method.

Running the Import
Navigate to Account Settings > Data Management > Import Data and select your import type. Upload your prepared CSV file. AcmeTech validates the file format and checks for errors before importing any records. If errors are detected, a validation report will be displayed showing the row number and error description for each issue. Correct the errors in your source file and re-upload. Once validation passes, click "Confirm Import" to proceed. Large imports processing more than 10,000 records are queued as background jobs. You will receive an email notification when the import completes.

Reviewing Import Results
After the import completes, review the import summary report showing the number of records successfully imported, updated, skipped, and failed. Download the detailed results file to see the status of each individual row. Failed rows are not imported and must be corrected and re-imported. Successfully imported records are immediately available in AcmeTech.

Troubleshooting Import Errors
Common import errors include duplicate records where a unique identifier already exists in AcmeTech (resolve by using the "Update" mode instead of "Create" mode), invalid field values that do not match the expected format or allowed values, and file encoding issues. If you receive an error that you cannot resolve using the validation report, contact AcmeTech Support and attach the import file and the error report for review.
